# Go Documentation Guidelines

1. **Use Godoc:** Follow standard Godoc conventions.
2. **Start with the Name:** Every exported (capitalized) identifier must have a doc comment starting with the name of the identifier (e.g., `// MyFunction does...`).
3. **Package Documentation:** Document the package itself with a comment immediately preceding the `package` clause.
4. **Examples:** Use testable examples (functions starting with `Example`) which will appear in Godoc.
