# Rust Documentation Guidelines

1. **Use Rustdoc:** Use `///` for outer documentation (documenting the item following it) and `//!` for inner documentation (documenting the enclosing item, like a module).
2. **Markdown Supported:** Make full use of Markdown inside Rustdoc comments.
3. **Doctests:** Code blocks in Rust documentation are automatically run as tests. Always ensure your code examples compile and pass.
4. **Sections:** Use standard sections like `# Examples`, `# Panics`, and `# Errors`.
