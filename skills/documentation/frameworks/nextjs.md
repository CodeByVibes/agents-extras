# Next.js Documentation Guidelines

1. **App Router Conventions:** Explicitly document the purpose of special files (`page.tsx`, `layout.tsx`, `loading.tsx`) if they contain complex logic.
2. **Client vs Server:** Clearly label in documentation if a component/utility is strictly for the server (`"use server"`) or client (`"use client"`).
3. **Data Fetching:** Document caching strategies used in data fetching functions to avoid unexpected stale data issues.
