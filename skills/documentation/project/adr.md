# Architecture Decision Records (ADR) Guidelines

When asked to write an ADR, you are recording a significant architectural decision.

1. **Use the Template:** Always use the [ADR Template](../templates/ADR.template.md).
2. **Naming Convention:** Name the file `XXXX-short-title.md` where `XXXX` is a monotonically increasing 4-digit number (e.g., `0001-use-postgresql.md`).
3. **Status:** The status should usually be `Proposed` when you first draft it.
4. **Context over Outcome:** Spend the most time detailing the *Context* (why are we making a decision?) and the *Consequences* (what happens now?). The *Decision* itself is usually short.
