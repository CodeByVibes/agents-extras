# Changelog Guidelines

When asked to update the changelog, adhere strictly to the "Keep a Changelog" format.

1. **Use the Template:** Follow the structure defined in [CHANGELOG Template](../templates/CHANGELOG.template.md).
2. **Unreleased Section:** Put all new changes into the `[Unreleased]` section. Only move them to a versioned section when instructed to perform a release.
3. **Categorize Changes:** Group changes into `Added`, `Changed`, `Deprecated`, `Removed`, `Fixed`, or `Security`.
4. **Audience:** Write changelog entries for *users* of the software, not just other developers. Avoid internal jargon unless necessary.
