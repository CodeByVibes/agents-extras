# README Guidelines

The README is the landing page for the project. When asked to create or update a README, follow these rules:

1. **Use the Template:** Always start with the [README Template](../templates/README.template.md).
2. **Be Concise:** The README should answer "What is this?" and "How do I start?" immediately. 
3. **Prerequisites First:** Never hide required dependencies deep in the installation section. List them at the top.
4. **Copy-Pasteable Commands:** Ensure all setup commands in the Installation block can be copy-pasted directly into a terminal and work.
5. **Keep Badges Minimal:** Include only necessary badges (CI status, License, Version). Do not clutter the top.
