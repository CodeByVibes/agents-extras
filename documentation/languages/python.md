# Python Documentation Guidelines

1. **Use Docstrings:** Follow PEP 257 for docstring conventions (`""" ... """`).
2. **Style:** Use the Google Style or NumPy Style for docstrings to ensure consistency across the project.
3. **Type Hints:** Rely on Python 3 type hinting (`def func(a: int) -> str:`) instead of documenting types inside the docstring. Use the docstring to explain *intent*.
4. **Exceptions:** Always document exceptions raised by a function using the `Raises:` section.
