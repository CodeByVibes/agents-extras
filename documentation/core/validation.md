# Documentation Validation Guidelines

Before concluding any documentation task, you must validate your work:

1. **Broken Links:** Check all internal relative links to ensure they point to existing files.
2. **Formatting:** Ensure markdown renders correctly. Do not nest Github Alerts (e.g., `> [!NOTE]`). Ensure lists are properly indented.
3. **Template Adherence:** Verify that no placeholder text (e.g., `[Project Name]`, `[Description]`) was accidentally left behind from a template.
